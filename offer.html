Контекст и проблема

Сейчас

Аналитические данные разрознены по источникам: Яндекс.Метрика, AppMetrica, Яндекс.Директ и продуктовая БД живут отдельно. Текущий контур построен маркетинговым подрядчиком (ePromo) под свои задачи ― на инфраструктуре beget, Python-скриптах и cron-джобах.

  • Нет единой воронки от трафика до повторной инвестиции
  • Нет когортного анализа и расчёта LTV
  • Нет контроля качества данных и SLA обновлений
  • Логика расчётов закрыта на стороне подрядчика

После проекта

Собственный DWH-контур на ClickHouse + Airflow, независимый от любого подрядчика. Все источники подключены через согласованные интеграции, данные трансформируются по прозрачным правилам.

  • Сквозная воронка: трафик → регистрация → идентификация → инвестиция → повтор
  • Когорты, Retention и LTV в одном интерфейсе
  • Единый глоссарий метрик, согласованный с бизнесом
  • Мониторинг, бэкапы, документация и регламент поддержки

CEO

CMO

Директор по маркетингу

  • Сквозная аналитика от клика до инвестиции по каналам
  • Точная стоимость привлечения (CAC) в разрезе UTM
  • Воронка с потерями на каждом шаге — понятно где чинить
  • Данные для A/B-тестов и оптимизации посадочных
CPO

Директор по продукту

  • Retention-кривые по когортам регистрации и первой инвестиции
  • Поведенческие паттерны: где и почему уходят пользователи
  • Событийная модель для оценки изменений в продукте
  • LTV-сегментация для приоритизации фич
CTO

Технический директор

  • Прозрачная архитектура: ClickHouse, Airflow, dbt — всё под вашим контролем
  • Данные и логика трансформаций у вас, не у подрядчика
  • Мониторинг, алерты, политика бэкапов
  • Документация и регламент для внутренней поддержки

Метрики, которые будут работать

Ниже — целевой набор под задачи инвестиционной платформы. Финальный словарь согласуется в ходе Discovery.

Метрика Что измеряет Зачем нужна
Воронка конверсии Трафик → регистрация → идентификация → первая инвестиция Видеть потери на каждом шаге и приоритизировать улучшения
CAC по каналам Стоимость привлечения инвестора в разрезе UTM / источника Перераспределять бюджет в прибыльные каналы
Retention D7 / D30 / D90 Доля инвесторов, вернувшихся к повторной инвестиции Оценивать удержание и влияние продуктовых изменений
LTV Пожизненная ценность клиента (суммарный доход / когорта) Принимать решения по unit-экономике и допустимому CAC
Time-to-invest Время от регистрации до первой инвестиции Находить точки ускорения онбординга
Идентификация: pass rate Доля успешных идентификаций (подача → результат) Диагностировать проблемы в процессе KYC
Avg deal size Средний размер инвестиции по сегментам Сегментировать аудиторию для персонализированных предложений
Повторные инвестиции Частота и объём reinvestment по когортам Понимать реальный жизненный цикл клиента
Качество данных Полнота, задержки загрузки, аномалии по источникам Обеспечивать достоверность всех расчётов

Этапы

Три параллельных стрима управляются единым трекером. Ниже — ключевые этапы с выходами.

1

Discovery

Доступы, интервью с бизнесом, инвентаризация источников, CJM, словарь метрик, карта событий.

Неделя 1–2 · Выход: ТЗ, event map
2

Architecture

Целевая схема хранилища, инфраструктура (ClickHouse + Airflow), SLA, безопасность, контрольный лист поставки.

Неделя 2–4 · Выход: готовая среда
3

Build

Подключение источников, слои данных (raw → staging → mart), витрины, BI-дашборды, контроль качества.

Неделя 4–9 · Выход: рабочие витрины
4

Launch

Мониторинг, резервирование, UAT с бизнесом, устранение замечаний, документация и передача в эксплуатацию.

Неделя 10 · Выход: промышленный запуск
