# DWH + BI Platform — Коммерческое предложение

**Исполнитель:** Business Metric · [business-metric.ru](https://business-metric.ru/) · ag@business-metric.ru · +7 917 530-25-89

---

## Задача

- Рост конверсии на каждом шаге воронки: трафик → регистрация → идентификация → первая инвестиция
- Рост Retention — возврат клиентов к повторным инвестициям
- Рост LTV — пожизненная ценность клиента как ключевой показатель unit-экономики

---

## Текущее состояние

Источники данных:
- Яндекс.Метрика — визиты, UTM, источники трафика
- AppMetrica — установки и сессии мобильного приложения
- Яндекс.Директ — рекламные расходы и кампании
- Продуктовая транзакционная БД — события и атрибуты пользователей

Текущий контур: `epromo_analytics` на MySQL (beget), ETL на Python/cron силами маркетингового подрядчика ePromo.

**Ограничения текущего решения:**
- Контур создан под задачи подрядчика, не под корпоративный DWH
- Логика трансформаций закрыта на стороне подрядчика, нет независимости
- Нет управления качеством данных, SLA, мониторинга, регламентов
- Модель данных не масштабируется под продуктовые сценарии

---

## Целевое решение

Независимый DWH-контур на базе ClickHouse + Airflow + dbt:
- Прозрачная архитектура слоёв: raw → staging → mart
- Регламентированная загрузка с контролем качества и SLA
- Витрины под задачи продукта и маркетинга
- BI-дашборды с согласованными метриками

---

## Метрики и аналитика

### Целевые метрики

| Метрика | Описание |
|---|---|
| Воронка конверсии | Трафик → регистрация → идентификация → первая инвестиция — конверсия и потери на каждом шаге |
| CAC по каналам | Стоимость привлечения инвестора в разрезе UTM / источника |
| Retention D7 / D30 / D90 | Доля вернувшихся к повторной инвестиции |
| LTV | Суммарная ценность клиента по когортам |
| Time-to-invest | Время от регистрации до первой инвестиции |
| Идентификация: pass rate | Доля успешных идентификаций (подача → результат) |
| Avg deal size | Средний размер инвестиции по сегментам |
| Повторные инвестиции | Частота и объём reinvestment по когортам |

### Карта пути клиента (CJM)

1. Трафик → посадочная страница
2. Посадочная страница → клик целевого действия
3. Регистрация
4. Подтверждение (телефон / почта / госуслуги)
5. Идентификация (подача)
6. Идентификация (результат)
7. Выбор продукта
8. Пополнение
9. Первая инвестиция
10. Повторная инвестиция

На каждом шаге: конверсия в следующий шаг (%), потери (%), время между шагами.

### Событийная модель

**Маркетинг:** `page_view`, `source/utm`, `scroll_depth`, `click_cta`

**Регистрация:** `signup_start`, `signup_complete`, `error_signup`

**Идентификация:** `ident_start`, `ident_step_N`, `ident_fail`, `ident_success`

**Пополнение:** `deposit_start`, `deposit_fail`, `deposit_success`

**Инвестиции:** `deal_view`, `deal_click`, `invest_start`, `invest_success`, `invest_cancel`

**Поведение:** `session_duration`, `screens_viewed`, `time_between_steps`

### Сегментация

- Канал привлечения (UTM): таргет, органика, рефералы, инфлюенс
- Устройство: iOS / Android / Web
- Новый / возвращающийся
- Лид / клиент
- Размер портфеля

## Состав работ

Три параллельных стрима, 10 недель.

### Стрим 1. Аналитика и постановка задач

**Результат:** ТЗ на целевую аналитику, словарь метрик, карта событий, макеты дашбордов, перечень задач на доработку трекинга.

Состав:
- Получение доступов к источникам и текущим отчётам
- Интервью с владельцами систем и бизнес-ответственными
- Фиксация метрик и правил расчёта
- Согласование границ первой версии
- Карта пути клиента (CJM) и карта событий
- ТЗ на доработки трекинга (веб, мобайл, UTM, tag container)

### Стрим 2. Инженерия данных и инфраструктура

**Результат:** Рабочий DWH-контур с автоматической загрузкой и расчётом витрин, набор проверок качества.

Состав:
- Целевая архитектура хранилища и схема потоков данных
- Настройка сервера, прав доступа, сетевых правил, хранения секретов
- Развёртывание ClickHouse (базы, таблицы, индексы, политика хранения)
- Развёртывание Airflow (планировщик, подключения, рабочие процессы)
- Подключение источников: Яндекс.Метрика, AppMetrica, Яндекс.Директ, продуктовая БД
- Реализация слоёв: raw → staging → mart
- Проверки качества: полнота, уникальность, своевременность, аномалии

### Стрим 3. Надёжность и ввод в эксплуатацию

**Результат:** Промышленный запуск, акт приёмки, регламентная модель эксплуатации.

Состав:
- Мониторинг загрузок, задержек, ошибок по каждому источнику
- Оповещения в мессенджер и почту, порядок реакции на инциденты
- Политика резервного копирования, контрольное восстановление
- Сквозное тестирование: источник → хранилище → витрина → дашборд
- Сверка ключевых показателей с согласованными расчётами
- Эксплуатационная документация, инструкции, регламент поддержки
- Передача прав и вводная сессия для ответственных сотрудников

| Этап | Недели | Результат |
|---|---|---|
| 1. Discovery | 1–2 | ТЗ, словарь метрик, event map, CJM |
| 2. Architecture | 2–4 | Готовая инфраструктура, архитектурная схема |
| 3. Build | 4–9 | Рабочие витрины, BI-дашборды |
